# 知识树使用说明

## 页面定位

`/relations` 页面用于维护父子关系（`PARENT_CHILD`），采用“浏览优先，编辑按需”的交互：

- 默认只展示知识树，不展示常驻编辑面板。
- 需要新增节点时，再打开右侧抽屉编辑。

`RELATED` 关联关系不在本页维护，统一在 `/nodes/[id]` 节点详情维护。

## 主流程

1. 浏览树结构

- 默认展开一级（根节点及其直接子节点）。
- 点击节点可选中；选中后可查看父子概览。

2. 新增节点

- 新增根节点：点击页面右上角 `新增根节点`。
- 新增子节点：
  - 选中节点后点击 `为当前节点新增子节点`；
  - 或在节点行 `hover` 后点击 `+`。
- 保存后抽屉自动关闭，树中自动定位并高亮新节点。

## 抽屉编辑器

### Markdown 编辑体验

- 桌面端：编辑区 + 实时预览并排展示。
- 移动端：支持 `编辑 / 预览` 切换。
- 提供常用插入按钮：标题、列表、代码块、引用。
- 快捷键：`Cmd/Ctrl + Enter` 提交。

### 草稿与离开保护

- 抽屉内容自动保存到本地 `localStorage`。
- 草稿按父节点分桶（根节点单独分桶）。
- 打开同一父节点抽屉时，如有草稿会提示“恢复 / 丢弃”。
- 关闭抽屉时，如有未保存内容会二次确认。
- 保存成功后会自动清除对应草稿。

## 二级操作（折叠区）

主界面默认隐藏以下操作，放在页面底部折叠区：

- 挂接已有节点为子节点（仅无父节点可挂接）。
- 解除当前父节点的子关系。

## 规则与错误

- 单父规则：每个子节点最多 1 条有效父边（`deletedAt IS NULL`）。
- 无环规则：父子关系必须保持 DAG。

常见错误：

- `CONFLICT: Child node already has a parent`：目标子节点已有父节点。
- `RELATION_CYCLE_DETECTED`：父子关系将形成环。
